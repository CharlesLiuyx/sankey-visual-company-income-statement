# Warner Bros. Discovery Q1 FY26 Icon Crop Validation

Dataset: `warner-bros-q1-fy26`

Source: `input/processed/warner-bros-q1-fy26.png`

Validation command:

```sh
python3 scripts/extract_icon_crops.py --spec input/icon-crop-specs/warner-bros-q1-fy26.json
```

## Accepted Crops

- `company-wordmark.png`: Warner Bros. Discovery shield and wordmark. The source flow fragment under the logo is removed with the spec-driven `transparentColors` rule; the accepted crop contains the company logo only.
- `business-studios-cluster.png`: Warner Bros., DC, and New Line Cinema marks for the Studios segment.
- `business-networks-cluster.png`: TBS, CNN, HGTV, TNT, Discovery, TLC, TVN, and Eurosport marks for the Networks segment.
- `business-streaming-cluster.png`: Max and discovery+ app icons for the Streaming segment.

All accepted crops are complete, visually centered, and free of unrelated labels, chart marks, connector fragments, watermarks, neighboring icon parts, and publisher/creator branding in the extracted crop panel of the validation sheets.

## Runtime Use

Image embedding mode is used because the segment clusters contain multiple brand-specific bitmap marks. Runtime copies were generated by the crop spec under:

`data/assets/raster-annotations/warner-bros/`

The d3 dataset explicitly opts in with `render.allowRasterAnnotations = true` and references only runtime copies, not `data/assets/icon-references/`.

## Skipped Source Marks

- Bottom-left "HOW THEY MAKE MONEY" publisher mark and mini Sankey logo.
- Bottom-center `appeconomyinsights.com`.
- Bottom-right App Economy Insights badge and wordmark.

These are publisher/creator attribution elements, not Warner Bros. Discovery company or business-segment semantics.
